# Security Policy

## Reporting a vulnerability

Please do **not** open public issues for security vulnerabilities.

Instead, report privately to the maintainers via your preferred secure channel.
If the org uses GitHub Security Advisories, use the “Report a vulnerability” button in the affected repository.

Include:
- Affected repository and component
- Steps to reproduce
- Impact assessment
- Suggested remediation (if any)

## Supported versions

Security fixes are applied to the latest stable releases of each repository. Long-term support (LTS) policies are defined per repo.

## Response process

1. Acknowledge receipt
2. Triage severity and scope
3. Develop and test a fix
4. Coordinate disclosure timeline
5. Publish advisory and patch
